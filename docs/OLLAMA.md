# Ollama: локальный перевод без API-ключей

Ollama — самый простой способ гонять LLM у себя на машине. pdftransl
общается с ней по OpenAI-совместимому API, так что никакой особой
интеграции не нужно: поставили, скачали модель — и переводите бесплатно
и без отправки статей в облако.

## Установка

**macOS** (проще всего):

```bash
brew install --cask ollama
```

…или скачайте приложение с [ollama.com](https://ollama.com/download) —
оно поселится в меню-баре и само запустит сервер. Если ставили через
brew без cask-версии, сервер запускается руками: `ollama serve`.

**Linux**:

```bash
curl -fsSL https://ollama.com/install.sh | sh
```

**Проверка, что сервер жив**:

```bash
curl http://localhost:11434/api/tags
# {"models":[...]} — значит всё в порядке
```

## Какую модель выбрать

Для перевода научных текстов нужна модель, которая хорошо держит
инструкции и знает оба языка. Семейство Qwen 2.5 — отличная рабочая
лошадка для пары en↔ru. Ориентируйтесь на память машины:

| RAM / VRAM | Модель | Команда | Комментарий |
|---|---|---|---|
| 8–16 ГБ | `qwen2.5:7b` | `ollama pull qwen2.5:7b` | быстро, качество «прилично» |
| 24–32 ГБ | `qwen2.5:14b` | `ollama pull qwen2.5:14b` | золотая середина — рекомендуем |
| 48+ ГБ | `qwen2.5:32b` | `ollama pull qwen2.5:32b` | близко к облачным моделям |

Альтернативы на попробовать: `llama3.1:8b` (хорош в en, слабее в ru),
`gemma2:9b`, `mistral-nemo:12b`. Скачивается всё одной командой
`ollama pull <имя>`, места на диске уходит примерно столько же,
сколько ГБ в размере модели.

На Apple Silicon (M1–M4) всё работает из коробки через Metal —
никаких драйверов. На Linux с NVIDIA поставится CUDA-вариант сам.

## Быстрый тест модели

```bash
ollama run qwen2.5:14b "Переведи на русский: The attention mechanism computes a weighted sum."
```

Если ответ осмысленный — модель готова.

## Подключение к pdftransl

`quickstart.sh` делает это сам: если Ollama запущена, он опрашивает её,
какие модели уже скачаны, и предлагает выбрать одну из них (рекомендация
под объём вашей памяти помечена в списке); если моделей нет — подскажет,
что скачать. Руками то же самое — три строчки в `.env`:

```env
PDFTRANSL_PROVIDER=ollama
PDFTRANSL_MODEL=qwen2.5:14b
PDFTRANSL_BASE_URL=http://localhost:11434/v1
```

…или на один запуск:

```bash
pdftransl translate статья.pdf --provider ollama --model qwen2.5:14b
```

Полезная связка — локальная модель с облачной подстраховкой на случай,
если Ollama упадёт или задохнётся:

```bash
pdftransl translate статья.pdf --provider ollama --fallback openrouter
```

## Тонкости и грабли

- **Контекстное окно.** По умолчанию Ollama часто режет контекст до
  2–4k токенов — для наших промптов с глоссарием и примерами этого
  мало. Поднимите глобально:

  ```bash
  export OLLAMA_CONTEXT_LENGTH=16384   # перед запуском ollama serve
  ```

  или создайте свой вариант модели:

  ```bash
  cat > Modelfile <<'EOF'
  FROM qwen2.5:14b
  PARAMETER num_ctx 16384
  EOF
  ollama create qwen2.5:14b-ctx16k -f Modelfile
  ```

- **Параллелизм.** Локальная модель обрабатывает запросы по одному.
  Ставить `--workers 8` смысла нет — оставьте 1–2, либо поднимите
  `OLLAMA_NUM_PARALLEL=2` (память ×2).

- **Мало RAM?** Возьмите квантованный вариант поменьше
  (`qwen2.5:7b-instruct-q4_K_M`) — качество проседает не так сильно,
  как кажется.

- **Модель «зависает» на первом запросе** — это загрузка весов в
  память, дальше будет быстрее. Держите Ollama запущенной.

- **Хочется качества повыше, а железо не тянет** — переводите локально,
  а LLM-ревью (`--score`) гоняйте облачной моделью: поставьте
  провайдера по умолчанию `ollama`, а в вычитке правьте руками — правки
  уйдут в память переводов и подтянут следующие документы.
