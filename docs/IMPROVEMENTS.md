# Предложения по улучшению (роадмап)

Текущая реализация — рабочая база. Ниже — улучшения в порядке
ожидаемой отдачи, с привязкой к существующим точкам расширения.

## 1. Качество парсинга

- **Альтернативные бэкенды**: добавить [marker](https://github.com/VikParuchuri/marker)
  (быстрее MinerU на CPU), [Nougat](https://github.com/facebookresearch/nougat)
  (сквозной OCR в LaTeX для сложной математики), Docling (IBM) и GROBID
  (точная сегментация библиографии). Интерфейс `ParserBackend` уже
  готов — один класс на бэкенд.
- **Ансамбль парсеров**: парсить двумя бэкендами и сверять — расхождения
  в формулах помечать для ручной проверки.
- **Валидация LaTeX**: прогонять извлечённые формулы через KaTeX/
  `pylatexenc` — синтаксически битые формулы отправлять в VLM на
  повторное распознавание по картинке страницы (кроп по bbox из
  `layout.json` MinerU).
- **Библиография**: детектировать секцию References и по умолчанию не
  переводить её (или переводить только аннотации) — сейчас она идёт
  общим текстом.
- **Двухколоночная вёрстка / сноски**: использовать `middle.json` MinerU
  для восстановления порядка чтения и привязки сносок.

## 2. Качество перевода

- **Кросс-сегментная согласованность терминологии**: автоматически
  извлекать термины из документа (LLM-проход по аннотации/введению),
  строить документный мини-глоссарий и подставлять его во все сегменты.
- **Контекстное окно**: передавать в промпт последние N переведённых
  предложений предыдущего сегмента — меньше швов на границах чанков.
- **Скользящее саммари документа** в system-промпте: модель понимает,
  о чём статья, и лучше разрешает многозначные термины.
- **Мультипровайдерный fallback**: цепочка провайдеров (локальный →
  облачный) при ошибках/лимитах; сейчас ретраи только внутри одного
  провайдера.
- **Батчинг и параллелизм**: сегменты независимы — переводить пулом
  потоков (для облака) с rate-limit'ом; выигрыш на длинных статьях
  кратный.
- **Structured outputs**: у провайдеров с JSON-schema (OpenAI, vLLM)
  запрашивать перевод в структурированном виде — исчезает класс ошибок
  «модель добавила комментарий».

## 3. Самоконтроль и метрики

- **Численная оценка качества**: COMET/BLEURT (referenceless-режимы,
  например CometKiwi) как скоринг каждого сегмента; порог — триггер
  ревью. Сейчас ревью запускается только по эвристикам.
- **Back-translation check**: обратный перевод дешёвой моделью +
  косинусная близость эмбеддингов оригинала и обратного перевода —
  ловит смысловые потери, которые не видят структурные валидаторы.
- **Рендер-проверка**: собрать итоговый Markdown → HTML (KaTeX) и
  убедиться, что все формулы рендерятся; ошибки рендера → отчёт.
- **Человек в цикле**: веб-UI со сравнением оригинал/перевод по
  сегментам, кнопкой «исправить» (правка уходит в TM как `human`) —
  API для этого уже есть (`add_correction`).

## 4. RAG и обучение

- **Векторный индекс**: заменить косинус в Python на sqlite-vec /
  pgvector / Qdrant (интерфейс `TranslationMemory` сохраняется) —
  нужно при >50k сегментов.
- **Доменные TM**: колонка `domain` уже по сути есть (`doc_id`);
  добавить фильтрацию поиска по области (физика/биология/ML) — меньше
  ложных few-shot примеров.
- **Дообучение локальной модели**: `tm export` уже даёт JSONL;
  добавить готовый рецепт LoRA (axolotl/unsloth) на накопленных парах —
  локальная модель со временем «выучивает» стиль и терминологию.
- **RAG по внешним источникам**: индексировать переведённые ранее
  статьи целиком и Wikipedia-глоссарии — подсказки для редких терминов.

## 5. Форматы вывода

- **Экспорт в PDF/DOCX** с сохранением вёрстки: Pandoc (md → docx/pdf
  через LaTeX), либо наложение перевода на исходную вёрстку по bbox
  (как делают BabelDOC/PDFMathTranslate) — сейчас выход только Markdown.
- **Двуязычный режим**: чередование абзацев оригинал/перевод — полезно
  для вычитки.
- **HTML-просмотрщик** с KaTeX и side-by-side сравнением.

## 6. Инфраструктура и бэкенд

- **Очередь с приоритетами и отменой**: сейчас Celery-таска монолитна;
  разбить на подзадачи (parse → translate → review) c чекпойнтами —
  перезапуск с места падения, отмена, параллельные стадии.
- **Стриминг прогресса**: SSE/WebSocket вместо поллинга (`on_stage`
  колбэк уже есть — осталось транслировать наружу).
- **Мультитенантность**: `user_id` в jobs/TM, квоты на токены,
  учёт стоимости по провайдерам (usage из ответов API уже доступен).
- **Docker-компоуз**: контейнеры web + worker + ollama + MinerU (GPU)
  для воспроизводимого развёртывания.
- **Кэш парсинга**: хэш PDF → готовый Markdown; повторная загрузка
  того же файла не тратит GPU-время.

## 7. Безопасность

- Ограничение размера/типа загружаемых PDF, антивирус-скан.
- Изоляция парсера (subprocess с таймаутом уже есть; добавить cgroup/
  контейнер — PDF-парсеры регулярно получают CVE).
- Секреты — только из окружения/vault, не из БД (уже так).
